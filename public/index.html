Como usar a API

Esta API expõe dados do GLPI em um formato plano, pronto para consumo por ferramentas de BI como o Power BI.

Endpoints principais

GET
/api/tickets/:status

Retorna chamados do GLPI filtrando por status (ex.: novo, atribuido, pausado, solucionado). 

GET https://perfillbi.onrender.com/api/tickets/atribuido
GET
/api/bi/tickets

Endpoint preparado para BI. Retorna uma lista plana de tickets com filtros de data opcionais.

  • startDate: data inicial (YYYY-MM-DD)
  • endDate: data final (YYYY-MM-DD)
GET https://perfillbi.onrender.com/api/bi/tickets?startDate=2026-03-01&endDate=2026-03-11

Formato de resposta (BI)

A resposta de /api/bi/tickets é sempre um array JSON plano:

[
  {
    "ticketId": 321,
    "titulo": "Camera offline",
    "status": "Atribuido",
    "dataCriacao": "2026-03-10",
    "dataFechamento": null,
    "...": "campos adicionais"
  }
]
| Header | Valor | | :--- | :--- | | **Authorization** | `Bearer ` |

Conectando o Power BI

  1. Abrir o Power BI Desktop.
  2. Clicar em Obter Dados > Web.
  3. Informar a URL do endpoint BI, por exemplo: https://perfillbi.onrender.com/api/bi/tickets?startDate=2026-03-01&endDate=2026-03-11.
  4. Configurar o Cabeçalho HTTP:
    • No Power BI, em "Obter Dados > Web", clique em Avançado.
    • Adicione um cabeçalho chamado Authorization.
    • O valor deve ser Bearer sua_chave_definida_no_env.
    • startDate representa a data inicial da consulta.
    • endDate representa a data final da consulta.
    • Para buscar todos os chamados, não informe os parâmetros de data.
    • Exemplo: https://SeuServidor/api/bi/tickets
  5. Confirmar e aguardar o carregamento dos dados.
  6. Modelar e criar os relatórios normalmente no Power BI.

O domínio oficial é https://perfillbi.onrender.com. Certifique-se de configurar o token API_AUTH_TOKEN no Render.
